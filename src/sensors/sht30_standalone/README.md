# SHT30 温湿度传感器测试程序

基于 Sensirion SHT30 数字温湿度传感器，通过 I2C 读取温度和相对湿度。

---

## 📁 文件

| 文件 | 说明 |
|------|------|
| `sht30_test.c` | C 源代码 |
| `sht30_test` | 交叉编译的 ARM 可执行文件 |
| `Makefile` | 编译脚本 |

## 🔌 硬件连接

| GY-SHT30-D | P9 引脚 | 说明 |
|------------|---------|------|
| VCC | Pin 1 (VCC_3V3) | 3.3V 供电（模块兼容 2.4~5.5V） |
| GND | Pin 6 (GND) | 地 |
| SDA | Pin 3 (I2C2_SDA) | I2C 数据 |
| SCL | Pin 5 (I2C2_SCL) | I2C 时钟 |

- I2C 总线: `/dev/i2c-2`
- I2C 地址: `0x44`（ADDR 引脚默认接 GND，接 VCC 则为 `0x45`）
- 模块已内置 10kΩ 上拉电阻，直连即可

## 🚀 使用方法

### 单次读取
```bash
./sht30_test -1 /dev/i2c-2
```

### 连续读取（默认 1s 间隔，Ctrl+C 停止）
```bash
./sht30_test /dev/i2c-2
```

### 每 5 秒读取一次
```bash
./sht30_test -i 5000 /dev/i2c-2
```

### 中精度快速读取
```bash
./sht30_test -m single-m -1 /dev/i2c-2
```

### 低精度超快读取 (~3ms)
```bash
./sht30_test -m single-l -1 /dev/i2c-2
```

### 周期模式（传感器自动测量，更省电）
```bash
./sht30_test -m p1-h -i 1000 /dev/i2c-2
```

### 高频采集 (10次/秒)
```bash
./sht30_test -m p10-l -i 100 /dev/i2c-2
```

### 详细模式（调试用）
```bash
./sht30_test -v -1 /dev/i2c-2
```

### 读取状态寄存器
```bash
./sht30_test --status /dev/i2c-2
```

### 软复位传感器
```bash
./sht30_test --reset /dev/i2c-2
```

### 加热器自检
```bash
./sht30_test --heater-on -1 /dev/i2c-2
```

## 📋 命令参数

| 参数 | 说明 |
|------|------|
| `-1` | 单次高精度读取 |
| `-m <mode>` | 测量模式（见下表） |
| `-i <ms>` | 读取间隔（毫秒, 默认 1000） |
| `-n <count>` | 单次模式读取次数 |
| `-a <addr>` | I2C 地址（默认 0x44） |
| `-v` | 详细模式（显示原始值、CRC） |
| `--heater-on` | 开启加热器（除湿/自检） |
| `--heater-off` | 关闭加热器 |
| `--status` | 读取状态寄存器 |
| `--reset` | 软复位传感器 |

| 模式名 | 命令码 | 测量时间 | 说明 |
|--------|--------|----------|------|
| `single-h` | 0x2C06 | ~15ms | 高重复精度（最准，默认） |
| `single-m` | 0x2C0D | ~6ms | 中重复精度 |
| `single-l` | 0x2C10 | ~3ms | 低重复精度（最快） |

## 📐 计算公式

```
T[°C] = -45 + 175 × (ST / 65535)       ST = 温度原始值 (16-bit)
RH[%] = 100 × (SRH / 65535)            SRH = 湿度原始值 (16-bit)

露点温度 (Magnus 近似):
γ = (a × T) / (b + T) + ln(RH/100)
Td = (b × γ) / (a - γ)                  a = 17.27, b = 237.7
```

## 🔐 CRC-8 校验

SHT30 每个 2 字节数据后跟 1 字节 CRC，多项式 `0x31`，初始值 `0xFF`。

本程序**自动执行 CRC 校验**，校验失败会打印警告。

## 🐛 故障排除

### 无法打开 I2C 设备
```bash
ls -l /dev/i2c*
i2cdetect -y -r 2
```
确认地址 0x44 出现。

### CRC 校验失败
- 检查接线，确保 SDA/SCL 接触良好
- 降低 I2C 速率（标准模式 100kHz）
- 确保电源稳定（3.3V）

### 读到 0 或不变的值
- 确认 ADDR 引脚接地（默认 0x44）
- 模块上电后等 1ms 再发送命令
- 尝试软复位 `./sht30_test --reset /dev/i2c-2`

### 温度偏高
- SHT30 自发热约 0.1~0.2°C（连续测量时）
- 过热检查是否误开了加热器：`./sht30_test --status /dev/i2c-2`
